<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted email content -> AI prompt | Email body/headers passed to Haiku/Sonnet; prompt injection risk |
| AI response -> ExecutedRule write | confidenceScore claimed by AI; written to DB without server-side validation beyond Zod's min/max |
| Conversation meta-rule -> classification prompt | Without guard, the meta-rule's instructions pollute the 8-category taxonomy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | AI prompt injection via email body | mitigate | Both Haiku and Sonnet calls go through `createGenerateObject({ promptHardening: { trust: "untrusted", level: "full" } })`. Already CLAUDE.md mandate; preserved across both tier calls (Task 1 acceptance). |
| T-03-15 | Spoofing | AI returns inflated confidenceScore to skip Sonnet | accept | Worst case: Haiku returns confidence=1.0 incorrectly and skips Sonnet, leading to misclassification. Mitigated by Phase 6 feedback loop (out of scope here). For Phase 3, `z.number().min(0).max(1)` clamps the value. Cost-of-mistake is bounded (single email mislabeled). |
| T-03-16 | Information Disclosure | confidenceScore leaks AI's certainty to a client | accept | Single-tenant; only Rebekah reads ExecutedRule via the digest. No external API surface exposes the column. |
| T-03-17 | Denial of Service | Two AI calls per email when escalation triggered | mitigate | Cost cap: $10/mo. Threshold of 0.8 expected to trigger ~10-15% escalation per RESEARCH cost model. Anthropic balance monitored (Plan 01 Task 2). If escalation rate spikes, Phase 6 feedback can tune the threshold. |
