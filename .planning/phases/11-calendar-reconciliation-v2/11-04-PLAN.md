Google Calendar API (googleapis-node):
- `client.events.get({ calendarId, eventId })` returns `{ data: { description?, htmlLink?, ... } }`. 404 throws `GaxiosError` with `code === 404` (or response.status === 404).
- `client.events.patch({ calendarId, eventId, requestBody: { description } })` performs a PATCH (partial update) — only the fields present in requestBody are modified. Other fields untouched on the server.

    ```ts
      | { ok: true }

    export async function patchEventDescription({
      input,
      logger,
    }: {
      input: {
        emailAccountId: string;
        eventId: string;          // Google event id of the OLD event being annotated
        appendText: string;       // The reschedule note, e.g. "[Possibly rescheduled? See https://calendar.google.com/...]"
      };
      logger: Logger;
    }): Promise<PatchEventDescriptionResult>;
    ```

    Behavior:
    1. Load `CalendarConnection` for `{ emailAccountId, provider: "google", isConnected: true }`. If none, return `{ ok: false, reason: "no-connection" }` and log warn with `{ emailAccountId }` only.
    2. Obtain a refreshed client via `getCalendarClientWithRefresh` using the same pattern createCalendarEvent does (including the `rawExpiresAt` narrowing block).
    3. Call `client.events.get({ calendarId: "primary", eventId: input.eventId })`. On 404 return `{ ok: false, reason: "event-not-found" }` and log warn with `{ emailAccountId, eventId }`. On other errors return `{ ok: false, reason: "api-error" }` and log error.
    4. Compute new description: `${existingDescription ?? ""}${existingDescription ? "\n\n" : ""}${input.appendText}`. If `existingDescription` already contains `input.appendText` verbatim (idempotency on retry), skip the patch and return `{ ok: true }`.
    5. Call `client.events.patch({ calendarId: "primary", eventId: input.eventId, requestBody: { description: newDescription } })`. On success return `{ ok: true }`. On error return `{ ok: false, reason: "api-error" }`.
    6. NEVER include `start`, `end`, `summary`, `location`, or `attendees` in the requestBody. (Google PATCH is partial — omitting these is what makes it safe.)
    7. Logging: only structured fields `{ emailAccountId, eventId, error }`. Never log the existing description, the appended text, the event title, or any other event payload.
    8. Wrap both API calls in try/catch — never throw out of this function (caller is the orchestrator, which respects failure-isolation per OPS-01).
