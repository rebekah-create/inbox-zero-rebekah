# Phase 8.5: Prompt Caching for Classification — Context

## Phase Boundary

**In scope:** Enable Anthropic prompt caching on the v1.0 Haiku classification prompt(s) used by `aiChooseRule`. Cache the constant-prefix portion (system text: instructions, rules list, user-info block, classification feedback) so repeat classifications pay ~10% of the input-token price on the cached prefix.

**Out of scope:**
- Touching the Phase 9 extraction prompt (separate phase will mirror the pattern)
- Refactoring `createGenerateObject` / `generateObject` plumbing beyond what's required to attach `cacheControl`
- Adding cache-hit metrics to the local DB (`saveAiUsage`); the success signal comes from the Anthropic Console dashboard
- Caching for non-Anthropic providers — gate on `provider === 'anthropic'` so caching never breaks other providers

## Locked Decisions

### D-01 — Cache target
The two classifier paths in `apps/web/utils/ai/choose-rule/ai-choose-rule.ts`:
- `getAiResponseSingleRule` (Tier 2 Haiku — the high-volume path)
- `getAiResponseMultiRule` (default model path)

Both build a `system` string and call `generateObject({ system, prompt, schema, ... })`. The constant prefix to cache is the full `system` text. The per-email `prompt` varies and is NOT cached.

### D-02 — Caching mechanism (AI SDK v6 + @ai-sdk/anthropic v3)
Pass the system content as a **`messages` array** instead of a plain `system` string, with the system message's content array marked:

```ts
const result = await generateObject({
  ...modelOptions,
  messages: [
    {
      role: 'system',
      content: [
        {
          type: 'text',
          text: systemText,
          providerOptions: {
            anthropic: { cacheControl: { type: 'ephemeral' } },
          },
        },
      ],
    },
    { role: 'user', content: prompt },
  ],
  schema,
});
```

Rationale: AI SDK's plain `system: string` does NOT support per-block `providerOptions`. Switching to `messages` keeps the same logical prompt but lets us attach `cacheControl` to the system block. The `prompt` becomes the first user message.

### D-03 — Provider gating
Only attach `cacheControl` when the resolved provider is `anthropic` (or `anthropic-vertex`). For OpenAI/OpenRouter/etc., fall back to today's `system: string` call shape. Caching markers on non-Anthropic providers are ignored by the SDK but the safer bet is to keep current call shape unchanged for non-Anthropic.

Read the provider from `modelOptions.provider` (compare against `Provider.ANTHROPIC` from `apps/web/utils/llms/config.ts`).

### D-04 — Minimum cacheable size
Anthropic requires ≥1024 tokens for Sonnet/Opus and ≥2048 tokens for Haiku 3 (≥1024 for newer Haiku) to actually cache. The system prompt with a real user's rules list comfortably exceeds 1024 tokens — no special guard needed. If the system is below threshold, Anthropic silently skips caching (no error). Acceptable.

### D-05 — Verification approach
- **Unit test:** Verify the request shape — when provider is `anthropic`, `generateObject` is called with a `messages` array whose first message has `cacheControl: { type: 'ephemeral' }` on the system text block.
- **Live verification:** After deploy, check the Anthropic Console usage dashboard at https://console.anthropic.com/settings/usage and confirm `cache_read_input_tokens > 0` within 24h. Recorded in CONTEXT decision log post-deploy, not blocking the PR.
- **No telemetry plumbing.** `result.providerMetadata.anthropic.usage` would expose `cacheCreationInputTokens` / `cacheReadInputTokens` but routing them through `saveAiUsage` is out of scope — Anthropic Console is the source of truth.

### D-06 — Documentation
Append a "Prompt Caching" section to `apps/web/utils/ai/choose-rule/NOTES.md` documenting:
- Which segments cache (full system block) vs vary (user prompt with email body)
- The cut point (one ephemeral marker on the trailing system block — the full system is one cache breakpoint)
- The Anthropic SDK pattern so Phase 9's extraction prompt mirrors it without re-discovery

## Specific Ideas

The cleanest call shape (works for both single-rule and multi-rule paths):

```ts
const baseRequest = { ...modelOptions, schema };
const result = await generateObject(
  isAnthropicProvider(modelOptions.provider)
    ? {
        ...baseRequest,
        messages: [
          {
            role: 'system',
            content: [
              {
                type: 'text',
                text: systemText,
                providerOptions: { anthropic: { cacheControl: { type: 'ephemeral' } } },
              },
            ],
          },
          { role: 'user', content: prompt },
        ],
      }
    : { ...baseRequest, system: systemText, prompt },
);
```
